Bias–Variance, PAC & VC Dimension

We can optimize and infer — but when does a model generalize? Learning theory answers with three tools: the bias–variance decomposition, the PAC framework for "probably approximately correct," and VC dimension, which turns "model capacity" into a number.

~22 min readNumPy/SciPy-verifiedPrereq: probability, MLE (Session 10)

A model that fits the training data perfectly can still be useless — it may have memorized noise. Learning theory is the study of the gap between training performance and true performance, and what controls it. This session builds the three pillars: the bias–variance decomposition (why error is U-shaped in complexity), PAC learning (a formal definition of "learnable" with a sample-complexity price tag), and VC dimension (a single number for a model class's capacity). Together they explain why more flexible isn't always better.

By the end you will be able to
  • Decompose expected error into bias² + variance + irreducible noise and read the U-shaped curve.
  • Explain underfitting vs overfitting as the bias–variance trade-off.
  • State the PAC criterion $(\varepsilon,\delta)$ and the finite-class sample-complexity bound.
  • Define shattering and VC dimension, and compute it for simple classifiers.
  • Read a generalization bound: test error $\le$ train error $+\ O\!\big(\sqrt{\text{VC}/n}\big)$.

Section 1The bias–variance decomposition

Suppose data is $y=f(x)+\epsilon$ with noise variance $\sigma^2$. For a model $\hat f$ trained on a random dataset, the expected squared error at a point decomposes exactly into three terms:

$$ \mathbb E\big[(y-\hat f(x))^2\big] = \underbrace{\big(\mathbb E[\hat f(x)]-f(x)\big)^2}_{\text{bias}^2} + \underbrace{\operatorname{Var}[\hat f(x)]}_{\text{variance}} + \underbrace{\sigma^2}_{\text{irreducible}}. $$
  • Bias² — error from the model being too simple to represent $f$ (systematic; doesn't vanish with more data).
  • Variance — sensitivity to the particular training sample (the model wobbles run to run).
  • Irreducible noise $\sigma^2$ — the floor; no model can beat it.

You cannot drive both bias and variance to zero at once with finite data — pushing one down usually pushes the other up. That tension is the trade-off.

Verified Fitting polynomials to noisy $\sin(2\pi x)$ (n=25) over 400 datasets: degree 1 → bias² $0.154$, var $0.024$ (underfit, bias dominates); degree 3 → $0.003 / 0.018$ (sweet spot, total $0.021$); degree 9 → bias² $0.19$ but variance $\mathbf{192}$; degree 12 → variance in the millions (E1). Bias falls, variance explodes — the classic picture.

Section 2The complexity dial — underfit ↔ overfit

Model complexity is a dial, and total error is U-shaped along it:

  • Too simple (left): high bias, low variance — underfitting. Train and test error both high.
  • Too complex (right): low bias, high variance — overfitting. Train error near zero, test error large.
  • The sweet spot (bottom): balanced, minimizing test error.

Regularization (ridge/lasso, dropout, early stopping) is precisely a tool for moving along this dial without changing the architecture — trading a little bias for a lot less variance.

Modern footnote: double descent Classical theory says error rises past the interpolation point. In very over-parameterized models (deep nets), pushing complexity further can make test error fall again — "double descent." The classical U still governs the under-parameterized regime; the second descent is an active research story, not a contradiction of the decomposition.

Section 3PAC learning — what "learnable" means

PAC (Probably Approximately Correct) makes "the model works" precise. A class is PAC-learnable if, given enough samples, an algorithm returns a hypothesis that is approximately correct (error $\le\varepsilon$) with high probability ($\ge1-\delta$):

$$ \Pr\big[\text{error}(\hat h)\le\varepsilon\big]\ \ge\ 1-\delta. $$

Two knobs: $\varepsilon$ (how approximate) and $\delta$ (how probably). The key quantity is sample complexity — how many examples $n$ you need. For a finite hypothesis class $\mathcal H$:

$$ n \ \ge\ \frac{1}{\varepsilon}\Big(\ln|\mathcal H| + \ln\tfrac{1}{\delta}\Big). $$

Read it: samples needed grow with the log of the class size (capacity) and shrink as you tolerate more error. Richer models need more data to pin down.

Verified $|\mathcal H|=1000,\ \varepsilon=0.1,\ \delta=0.05\Rightarrow n\ge100$; $|\mathcal H|=10^6,\ \varepsilon=0.05,\ \delta=0.01\Rightarrow n\ge369$ (E3). Note the $\ln$: a thousand-fold bigger class needs only a few times more data.

Section 4VC dimension — capacity as a number

Most useful hypothesis classes are infinite (all linear classifiers), so $|\mathcal H|$ is useless. VC dimension measures capacity for infinite classes via shattering: a set of points is shattered by $\mathcal H$ if, for every possible labeling of those points, some hypothesis in $\mathcal H$ realizes it.

$$ \text{VC}(\mathcal H) = \text{size of the largest set that } \mathcal H \text{ can shatter}. $$

Some classic values:

Hypothesis classVC dimension
Thresholds on $\mathbb R$ ($x>t$)1
Intervals on $\mathbb R$2
Linear classifiers in $\mathbb R^d$$d+1$
1-nearest-neighbour$\infty$ (can shatter anything)
Verified Linear classifiers in 2D: three non-collinear points are shattered — all $2^3=8$ labelings are linearly separable ($8/8$). Four points cannot be — the two XOR labelings are not separable ($14/16$), so VC $=3=d+1$ (E2). This is the geometric fact behind "a line can't solve XOR."

Section 5The generalization bound — capacity controls the gap

VC dimension turns into a guarantee. With probability $\ge1-\delta$, for all $h\in\mathcal H$:

$$ \underbrace{\text{error}_{\text{test}}(h)}_{\text{true risk}} \ \le\ \underbrace{\text{error}_{\text{train}}(h)}_{\text{empirical risk}} + O\!\left(\sqrt{\frac{\text{VC}(\mathcal H)\,\ln n + \ln\tfrac1\delta}{n}}\right). $$

The gap between test and train error is bounded by (roughly) $\sqrt{\text{VC}/n}$. Two levers: less capacity (smaller VC) or more data (larger $n$) both shrink the gap. This is the theoretical backbone of everything you already do:

  • Occam's razor: prefer the simplest model that fits — lower VC, tighter bound.
  • Structural risk minimization: explicitly trade empirical error against a capacity penalty (regularization is SRM in practice).
  • "Get more data": the $\sqrt{1/n}$ decay is why data volume beats cleverness so often.
The one-sentence theory of generalization A model generalizes when its capacity is small relative to the amount of data — fit the data with the least flexibility that suffices, and the train/test gap stays controlled.

Section 6Doing it in NumPy ✓ outputs verified

import numpy as np
rng = np.random.default_rng(0)
f = lambda x: np.sin(2*np.pi*x)
xt = np.linspace(0.05, 0.95, 50); ft = f(xt)

def bias_var(deg, n=25, sigma=0.3, trials=400):
    preds = np.zeros((trials, len(xt)))
    for t in range(trials):
        x = rng.uniform(0,1,n); y = f(x) + rng.normal(0,sigma,n)
        preds[t] = np.polyval(np.polyfit(x,y,deg), xt)
    bias2 = np.mean((preds.mean(0) - ft)**2)
    var   = np.mean(preds.var(0))
    return bias2, var

for d in (1,3,9):
    print(d, np.round(bias_var(d), 4))   # 1:[.154 .024]  3:[.003 .018]  9:[.19 192.]

# PAC sample complexity for a finite hypothesis class
H, eps, delta = 1000, 0.1, 0.05
print(np.ceil((np.log(H)+np.log(1/delta))/eps))   # 100
1 [0.154 0.0244] 3 [0.0031 0.0177] 9 [0.1915 192.25 ] 100.0
The practical read Watch the train–test gap. Small gap + high error = underfit (raise capacity). Large gap = overfit (lower capacity or add data). That gap is the generalization bound, made concrete.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Bias–variance is exact for squared loss; for other losses (0–1, cross-entropy) the clean three-term split doesn't hold, though the intuition carries.
  • Shattering needs "for every labeling": finding one labeling a class can realize proves nothing; VC requires all $2^k$ labelings of some $k$-set.
  • VC bounds are loose: they're worst-case and often pessimistic; real models generalize better than VC predicts (hence margin/Rademacher bounds — Session 14).
  • High VC ≠ doomed: huge-capacity nets generalize well with enough data and implicit regularization (double descent).
Trap 1 · Judging a model by training errorZero training error can mean perfect learning or pure memorization. Only the held-out/test error (the gap) reveals which.
Trap 2 · "More parameters = more capacity = worse"Not necessarily — capacity depends on the class geometry (VC/margins), not raw parameter count. Over-parameterized + regularized can generalize beautifully.
Trap 3 · Confusing shatterable-somewhere with shatters-allVC is the largest set that can be shattered; you need only one such set, but that set's every labeling must be realizable.

Golden rules

  1. Error $=$ bias² + variance + irreducible noise; you can't zero the first two together.
  2. Test error is U-shaped in complexity: simple → high bias, complex → high variance.
  3. PAC: error $\le\varepsilon$ w.p. $\ge1-\delta$; finite-class $n\ge\tfrac1\varepsilon(\ln|\mathcal H|+\ln\tfrac1\delta)$.
  4. VC dimension = largest shatterable set; linear in $\mathbb R^d$ is $d+1$.
  5. Generalization gap $\lesssim\sqrt{\text{VC}/n}$ — shrink it with less capacity or more data.
  6. Regularization = moving down the complexity dial; watch the train–test gap.

PracticeExercises

Measure the trade-off yourself.

E1medium

Empirically compute bias² and variance for polynomial degrees 1, 3, 9 fitting $\sin(2\pi x)$, and identify the sweet spot.

Show solution

Using the bias_var function: deg 1 → $[0.154, 0.024]$ (bias-limited), deg 3 → $[0.003, 0.018]$ (lowest total ≈ 0.021), deg 9 → $[0.19, 192]$ (variance-limited). Degree 3 is the sweet spot for this noise/size.

E2hard

Show a linear classifier shatters 3 non-collinear points in 2D but not 4, using a linear-separability test.

Show solution
from scipy.optimize import linprog
from itertools import product
def separable(X,y):
    ys=np.where(np.array(y)==1,1,-1); n=len(X)
    A=-(ys[:,None]*np.hstack([X,np.ones((n,1))])); b=-np.ones(n)
    return linprog(np.zeros(3), A_ub=A, b_ub=b, bounds=[(-100,100)]*3).success
tri=np.array([[0,0],[1,0],[0.4,1]])
sum(separable(tri,l) for l in product([0,1],repeat=3))   # 8/8 -> shatters

3 points: 8/8 labelings separable ⇒ shattered ⇒ VC ≥ 3. The square's XOR labelings fail (14/16) ⇒ can't shatter 4 ⇒ VC $= 3 = d+1$.

E3easy

Use the PAC bound to find $n$ for $|\mathcal H|=10^6$, $\varepsilon=0.05$, $\delta=0.01$.

Show solution
np.ceil((np.log(1e6)+np.log(1/0.01))/0.05)   # 369

$n\ge\tfrac{1}{0.05}(\ln10^6+\ln100)=\tfrac{1}{0.05}(13.8+4.6)\approx369$. The $\ln$ keeps it modest even for a million hypotheses.

E4medium

Explain why 1-nearest-neighbour has infinite VC dimension and what that implies about its training error.

Show solution

1-NN can realize any labeling of any finite point set (each point is its own nearest neighbour), so it shatters sets of every size ⇒ VC $=\infty$. Consequence: its training error is always 0 (each point predicts its own label), which tells you nothing — its generalization comes entirely from data density and the distance metric, not a small hypothesis class.

E5medium

You observe train error 2%, test error 30%. Diagnose and prescribe using bias–variance.

Show solution

Large train–test gap (28 points) with low train error ⇒ high variance / overfitting. Prescriptions: reduce capacity (smaller/regularized model, fewer features), add regularization (L2, dropout, early stopping), or get more data ($\sqrt{1/n}$ shrinks the gap). If instead both errors were ~30%, that'd be high bias (underfit) — raise capacity.

PayoffWhere this lives in real ML

  • Model selection: cross-validation is the empirical search for the bottom of the U.
  • Regularization: ridge/lasso/dropout/early-stopping all move down the complexity dial (structural risk minimization).
  • Diagnosing training: the train–test gap tells you bias vs variance and what to fix.
  • Data strategy: the $\sqrt{1/n}$ law quantifies how much more data buys.

InterviewQuestions you should be able to answer

Q1 · State the bias–variance decomposition.
$\mathbb E[(y-\hat f)^2]=\text{bias}^2+\text{variance}+\sigma^2$. Bias = systematic error of the average model; variance = sensitivity to the training sample; $\sigma^2$ = irreducible noise.
Q2 · Overfitting in bias–variance terms?
Low bias, high variance: the model fits training noise, so training error is low but test error high (large gap).
Q3 · What does PAC-learnable mean?
With enough samples, an algorithm returns a hypothesis with error $\le\varepsilon$ with probability $\ge1-\delta$ — "probably approximately correct," with a sample-complexity guarantee.
Q4 · Define VC dimension and give one value.
The size of the largest point set the class can shatter (realize all labelings of). Linear classifiers in $\mathbb R^d$ have VC $=d+1$; in 2D that's 3 (a line can't do XOR).
Q5 · How does capacity relate to generalization?
Test error $\le$ train error $+O(\sqrt{\text{VC}/n})$: the gap grows with capacity and shrinks with data. Less capacity or more data ⇒ better generalization.

◆ Concept Card
Generalization = fit the data with the least flexibility that suffices.

Reusable idea: the gap between training and true error is governed by capacity vs data.

  • Error = bias² + variance + noise; complexity trades bias for variance, so test error is U-shaped.
  • PAC makes "learnable" precise: error $\le\varepsilon$ w.p. $\ge1-\delta$, needing $n\gtrsim\tfrac1\varepsilon\ln|\mathcal H|$ samples.
  • VC dimension = largest shatterable set (linear in $\mathbb R^d$: $d+1$) — capacity for infinite classes.
  • Gap $\lesssim\sqrt{\text{VC}/n}$: shrink it with less capacity or more data — that's what regularization and "get more data" do.
The single hook: a perfect training fit means nothing on its own — what matters is capacity relative to data. Every regularizer, every cross-validation split, every "we need more data" is a move in the bias–variance / capacity game this session formalizes.
