The Math of Attention & Transformers

Strip away the mystique and attention is one line: a softmax over scaled dot products, times a matrix of values. It's a differentiable, soft dictionary lookup — and it assembles from the dot products, softmax, and matrix calculus you already have.

~22 min readNumPy-verifiedPrereq: dot product, softmax, matrix calculus

Attention is the operation that ended the RNN era and powers every large language model — yet mathematically it's assembled entirely from pieces you've already met: dot products (similarity, Session foundational-02), softmax (Session 18/10), and matrix multiplication. This session writes the whole thing as one formula, explains the two design choices that make it work in practice (the $\sqrt d$ scaling and masking), and locates its notorious cost. By the end, “self-attention” should read as a soft, learnable lookup — nothing more, nothing magic.

Notation reference
$Q,K,V$Queries, keys, values — each a matrix of $n$ rows (tokens) × $d$ features. $d$Head dimension (length of each query/key vector). $n$Sequence length (number of tokens). $\mathrm{softmax}$Row-wise normalization to a probability distribution over keys. mask$-\infty$ added to forbidden scores (e.g. future positions) before softmax.

Section 1The attention operation, in one line

Each token emits a query (what it's looking for), a key (what it offers), and a value (what it passes on). A query scores every key by dot product, those scores become weights via softmax, and the output is the weighted average of the values:

$$ \mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt d}\right) V. $$

Read it right-to-left in three steps: (1) $QK^\top$ — every query · every key = an $n\times n$ table of similarities; (2) $\mathrm{softmax}$ each row — turn similarities into weights that sum to 1; (3) multiply by $V$ — each output row is a weighted blend of value vectors, weighted by how well its query matched each key. That's it — a soft, content-based lookup.

Analogy Attention is a soft dictionary lookup. A normal dictionary needs an exact key and returns one value. Attention compares your query to every key by similarity, turns those similarities into weights, and returns a blend of all the values — mostly the best-matching one, a little of the others. It's a fuzzy, differentiable key–value store, which is exactly why it can be trained by gradient descent.
Verified A from-scratch implementation confirms the mechanics: every row of the softmax weights sums to $1$, and the output has shape $n\times d$ — each row a convex combination of the $n$ value rows (E1). No loops, no recurrence: all pairwise interactions in two matrix multiplies.

Section 2Why divide by $\sqrt d$

The scaling isn't cosmetic. A dot product of two $d$-dimensional vectors with unit-variance entries has variance $\approx d$, so the raw scores $QK^\top$ have standard deviation $\approx\sqrt d$. Feed scores that large into softmax and it saturates — one entry rounds to 1, the rest to 0 — which kills the gradient (a saturated softmax has near-zero slope, Session 10's Fisher/curvature story). Dividing by $\sqrt d$ rescales the scores back to variance $\approx1$, keeping softmax in its responsive, trainable regime.

Verified Raw dot-product std tracks $\sqrt d$ — $2.76$ at $d{=}8$, $7.96$ at $d{=}64$, $22.4$ at $d{=}512$ — and dividing by $\sqrt d$ pulls it to $\approx1.0$ every time (E2). And an unscaled large-logit softmax collapses: $\mathrm{softmax}([2,1,0])=[.67,.25,.09]$ but $\mathrm{softmax}(8\cdot[2,1,0])=[1,0,0]$ (E3) — saturated, no gradient.

Section 3Multi-head attention

One attention operation captures one kind of relationship. Multi-head attention runs $h$ of them in parallel, each with its own learned projections of $Q,K,V$ into a smaller subspace, then concatenates the results:

$$ \mathrm{MHA} = \big[\,\mathrm{head}_1;\dots;\mathrm{head}_h\,\big]\,W^O, \qquad \mathrm{head}_i=\mathrm{Attention}(QW_i^Q, KW_i^K, VW_i^V). $$

Different heads learn to attend to different things — one to the previous token, one to the subject of the sentence, one to matching brackets. It's an ensemble of soft lookups, each looking at the sequence through a different lens, fused by the output projection $W^O$. The total cost is roughly the same as single-head with full dimension, since each head works in $d/h$ dimensions.

Section 4Masking & positional encoding

Two structural facts about the raw operation need fixing.

Masking. Attention as written lets every token see every other. For autoregressive generation (predict the next token) a token must not peek at the future, so we add $-\infty$ to the scores of future positions before softmax — they get weight 0. That causal mask makes a GPT; no mask makes a bidirectional encoder (BERT).

Verified With a causal mask, the attention rows are lower-triangular: position 1 attends only to itself ($[1,0,0]$), position 2 to the first two ($[0.71,0.29,0]$), position 3 to all past ($[0.16,0.17,0.67]$) (E4) — the future is invisible.

Position. Attention is a set operation — permute the tokens and the outputs just permute; it has no sense of order. So order must be injected separately, via positional encodings (sinusoidal, learned, or rotary/RoPE) added to or mixed into the token representations. Without them “dog bites man” and “man bites dog” are identical to the model.

Section 5Cost & the transformer block

The $n\times n$ score matrix is attention's power and its curse. Computing $QK^\top$ and applying it to $V$ is $O(n^2 d)$ time and — crucially — $O(n^2)$ memory for the weights. That quadratic-in-sequence-length scaling is why long context is expensive and why a whole industry of fixes exists: FlashAttention (same math, never materializes the full $n\times n$ matrix — an IO-aware rewrite), and linear/sparse attention (approximate the softmax to get $O(n)$).

A full transformer block wraps attention with the rest of Pillars I–IV: multi-head attention, then a position-wise feed-forward network, each with a residual connection and layer normalization. Stack these, add positional encodings, and you have the architecture behind modern AI — attention doing the mixing across tokens, the feed-forward doing the per-token computation.

Section 6Doing it in NumPy ✓ outputs verified

import numpy as np
def softmax(z, axis=-1):
    z = z - z.max(axis, keepdims=True); e = np.exp(z)
    return e / e.sum(axis, keepdims=True)

def attention(Q, K, V, mask=None):
    d = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d)               # n x n similarities, scaled
    if mask is not None: scores = np.where(mask, -np.inf, scores)
    A = softmax(scores)                          # rows sum to 1
    return A @ V, A                             # weighted blend of values

rng = np.random.default_rng(0); n, d = 4, 8
Q,K,V = (rng.standard_normal((n,d)) for _ in range(3))
out, A = attention(Q, K, V)
print(A.sum(1))                              # [1. 1. 1. 1.]

# causal mask: add -inf to future positions before softmax
mask = np.triu(np.ones((3,3)), 1).astype(bool)
X = rng.standard_normal((3,8))
_, Ac = attention(X, X, X, mask)             # self-attention, masked
print(np.round(Ac, 2))                     # lower-triangular weights
[1. 1. 1. 1.] # attention weights are proper distributions # causal rows: [1,0,0], [0.71,0.29,0], [0.16,0.17,0.67]
Computational & memory cost Attention is $O(n^2 d)$ time and $O(n^2)$ memory — quadratic in sequence length $n$. That's why doubling context roughly quadruples the attention cost, and why long-context models rely on FlashAttention (an IO-aware exact algorithm that never stores the full $n\times n$ matrix) or linear/sparse attention (approximate softmax for $O(n)$). Multi-head doesn't add cost — each head runs in $d/h$ dimensions.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Attention is permutation-equivariant: no positional encoding ⇒ the model can't tell word order. Always inject position.
  • Softmax saturation: without $\sqrt d$ scaling (or with extreme logits), gradients vanish — attention stops learning.
  • Mask before softmax, with $-\infty$ (not 0 after) — masking after softmax breaks the normalization.
  • Numerical stability: subtract the row max before exponentiating (the code does) or large scores overflow.
Trap 1 · Forgetting positional informationRaw attention is order-blind; “dog bites man” = “man bites dog” without positional encodings. This is a classic source of silently broken sequence models.
Trap 2 · Dropping the $\sqrt d$ scaleUnscaled scores grow with dimension and saturate softmax; training stalls with near-zero attention gradients. Never remove the scaling.
Trap 3 · Assuming attention is cheapThe $O(n^2)$ memory is the real limit on context length. Naïvely quadrupling the context can OOM; use FlashAttention or a sub-quadratic variant.

Golden rules

  1. Attention $=\mathrm{softmax}(QK^\top/\sqrt d)\,V$ — a soft, weighted value lookup; rows sum to 1.
  2. Divide by $\sqrt d$ to keep score variance ≈1 and softmax un-saturated.
  3. Multi-head = parallel attentions in subspaces, capturing different relations.
  4. Mask (add $-\infty$) for causal generation; add positional info since attention is order-blind.
  5. Cost is $O(n^2 d)$ time, $O(n^2)$ memory — the long-context bottleneck (FlashAttention / linear attention).
  6. A transformer block = multi-head attention + feed-forward + residual + layer norm.

PracticeExercises

Implement the pieces and see the design choices bite.

E1medium

Implement scaled dot-product attention and confirm the weight rows sum to 1 and the output is a convex blend of $V$.

Show solution
def attention(Q,K,V):
    d=Q.shape[-1]; A=softmax(Q@K.T/np.sqrt(d)); return A@V, A
_,A=attention(*[rng.standard_normal((4,8)) for _ in range(3)])
A.sum(1)   # [1 1 1 1]  -> each output row is a weighted average of value rows
E2medium

Empirically show the std of raw dot products grows like $\sqrt d$ and that dividing by $\sqrt d$ normalizes it.

Show solution
for d in (8,64,512):
    q=rng.standard_normal((2000,d)); k=rng.standard_normal((2000,d))
    dots=np.sum(q*k,1); print(d, dots.std(), (dots/np.sqrt(d)).std())
# std ~ sqrt(d) (2.8, 8, 22.6); after scaling ~ 1.0 each
E3easy

Show that scaling logits up by 8× saturates softmax to a near one-hot vector.

Show solution
softmax(np.array([2.,1.,0.]))     # [.67 .25 .09]
softmax(8*np.array([2.,1.,0.]))   # [1 0 0]  saturated, ~zero gradient

Large logits push softmax to the vertex of the simplex where its Jacobian ($\mathrm{diag}(p)-pp^\top$, Session 05) is nearly zero — no gradient to learn from. The $\sqrt d$ scale prevents exactly this.

E4medium

Build a causal mask and confirm each position attends only to itself and earlier positions.

Show solution
s=rng.standard_normal((3,3)); mask=np.triu(np.ones((3,3)),1).astype(bool)
softmax(np.where(mask,-np.inf,s))   # lower-triangular: [1,0,0],[.71,.29,0],[.16,.17,.67]

Adding $-\infty$ to future positions before softmax gives them weight 0, so the output at each position depends only on the past — the definition of an autoregressive model.

E5hard

Show attention is permutation-equivariant (no position sense): permuting the input rows just permutes the output rows.

Show solution
X=rng.standard_normal((4,8)); P=np.eye(4)[[2,0,3,1]]   # a permutation
o1,_=attention(X,X,X); o2,_=attention(P@X,P@X,P@X)
np.allclose(P@o1, o2)   # True -> output permutes with the input

Reordering tokens simply reorders the outputs — attention has no built-in notion of position, which is why positional encodings must be added. “dog bites man” and “man bites dog” are indistinguishable without them.

PayoffWhere this lives in real ML

  • Every LLM: stacked self-attention + feed-forward blocks are the transformer — GPT, BERT, Llama, Claude.
  • Cross-attention: queries from one sequence, keys/values from another — translation, retrieval-augmented generation, image captioning.
  • Vision & multimodal: ViT, CLIP — patches or modalities as tokens.
  • Efficiency research: FlashAttention, linear/sparse attention, and long-context tricks all target the $O(n^2)$ cost.

InterviewQuestions you should be able to answer

Q1 · Write scaled dot-product attention and explain each part.
$\mathrm{softmax}(QK^\top/\sqrt d)V$: $QK^\top$ scores each query against each key; softmax turns scores into weights summing to 1; multiplying by $V$ returns a weighted blend of values — a soft lookup.
Q2 · Why the $\sqrt d$ scaling?
Dot products of $d$-dim vectors have std $\approx\sqrt d$; unscaled, softmax saturates and gradients vanish. Dividing by $\sqrt d$ keeps score variance ≈1 and softmax responsive.
Q3 · What does multi-head attention add?
Parallel attention in different learned subspaces, so different heads capture different relationships (syntax, coreference, position); results are concatenated and projected.
Q4 · Causal vs bidirectional masking?
Causal mask (add $-\infty$ to future scores) makes an autoregressive model (GPT); no mask lets every token see all others (BERT-style encoder).
Q5 · What's attention's computational cost and how is it mitigated?
$O(n^2 d)$ time and $O(n^2)$ memory in sequence length. FlashAttention avoids materializing the full matrix (exact, IO-aware); linear/sparse attention approximates softmax for $O(n)$.

◆ Concept Card
Attention is a soft, differentiable dictionary lookup.

Reusable idea: the operation behind modern AI is one line built from dot products and softmax.

  • $\mathrm{softmax}(QK^\top/\sqrt d)\,V$ — score every key by a query, normalize to weights, return a weighted blend of values.
  • Divide by $\sqrt d$ so scores stay variance-1 and softmax doesn't saturate (dead gradients).
  • Multi-head = parallel lookups in subspaces; mask for causality; add position because attention is order-blind.
  • Cost $O(n^2 d)$ / $O(n^2)$ memory — the quadratic bottleneck FlashAttention and linear attention exist to beat.
The single hook: there is no magic — attention replaces “pass memory down a chain” (RNNs) with “every token looks at every token and blends what's relevant,” a parallel, distance-free, differentiable lookup. Stack it with feed-forward, norm, and residuals and you have the transformer.
